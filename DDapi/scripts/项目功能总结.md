# 小白机器人 - 项目功能总结

## 项目概述

「小白」是一个基于钉钉平台的 AI 聊天机器人，集成了 **LangGraph 图智能体**、MiMo 大模型、Redis 记忆系统、Milvus 向量数据库，支持日程管理、签到查询、消息发送、知识库检索等功能。

## 核心架构

### LangGraph 图智能体

项目使用 **LangGraph** 替代了原有的 LangChain AgentExecutor，获得更好的流程控制和状态管理。

```
用户消息
    │
    ▼
┌─────────────┐
│  agent_node  │  ← LLM决策是否调用工具
└──────┬──────┘
       │
       ├──────────────────┐
       ▼                  ▼
┌─────────────┐    ┌─────────────┐
│  tools_node  │    │ save_memory  │
│  (执行工具)  │    │  (保存对话)  │
└──────┬──────┘    └──────┬──────┘
       │                  │
       ▼                  ▼
   回到agent_node        END
```

**核心组件：**
- `StateGraph`: 状态图定义工作流
- `ToolNode`: 工具执行节点
- `RedisSaver`: Redis Checkpointer 持久化对话状态
- `add_messages`: 消息自动追加

## 文件功能详解

### 1. `main.py` - 启动入口

| 功能 | 说明 |
|------|------|
| 钉钉连接 | 使用 `dingtalk_stream` 建立长连接 |
| 消息注册 | 注册 `PrivateMessageHandler` 处理私聊消息 |
| 永久运行 | `start_forever()` 保持服务常驻 |

**启动命令**: `python main.py`

---

### 2. `config.py` - 配置文件

| 配置项 | 说明 |
|--------|------|
| `Model_id` | 模型选择开关（0=MiMo API, 1=本地 Ollama） |
| `llm` | LLM 模型实例（MiMo 或 qwen2.5） |
| `redis_url` | Redis 连接地址 |
| `system_text` | 系统提示词（小白人设 + 工具说明） |
| `client_id` | 钉钉应用 AppKey |
| `client_secret` | 钉钉应用 AppSecret |
| `accessToken` | 钉钉访问令牌（动态刷新） |
| `userid` | 用户 ID |
| `unionid` | 用户 UnionID（日程 API 需要） |
| `robot_code` | 机器人代码 |
| `PDF_BASE_PATH` | PDF文档目录路径 |
| `MILVUS_URI` | Milvus 连接地址 |
| `COLLECTION_NAME` | Milvus 集合名称 |

**LLM 切换**:
- `Model_id=0`: MiMo API（`https://token-plan-cn.xiaomimimo.com/v1`）
- `Model_id=1`: 本地 Ollama（`qwen2.5`）

---

### 3. `tools/graph_agent.py` - LangGraph 图智能体（核心）

| 组件 | 功能 |
|------|------|
| `AgentState` | 状态定义（messages, user_id, memory_content） |
| `agent_node` | Agent 节点：LLM 决策是否调用工具 |
| `save_memory_node` | 保存本轮对话到记忆系统 |
| `ToolNode` | 工具执行节点（LangGraph 预置） |
| `RedisSaver` | Redis Checkpointer 持久化对话状态 |
| `build_graph()` | 构建状态图工作流 |
| `chat()` | 处理一轮对话的主函数 |

**状态流转**:
```
START → agent_node → (工具调用?) → tools_node → agent_node
                  → (无工具调用?) → save_memory_node → END
```

**调用流程**:
```
用户消息 → chat() → 获取记忆上下文 → 图执行 → 保存对话 → 返回回复
```

### 5. `tools/processMsg.py` - 消息处理器

| 组件 | 功能 |
|------|------|
| `PrivateMessageHandler` | 继承 `dingtalk_stream.ChatbotHandler` |
| `_is_reset_command()` | 检测会话重置指令（新对话/重新开始/清除记忆/重置） |
| `process()` | 处理私聊消息的异步方法 |

**处理流程**:
```
接收消息 → 检测重置指令 → 调用 graph_agent.chat() → 发送回复
```

**会话重置**: 支持关键词触发（新对话/重新开始/清除记忆/重置），重置后保留用户记忆但清空对话历史。

---

### 6. `tools/memory_store.py` - 记忆存储

#### Redis 数据结构

| Key 格式 | 类型 | 说明 |
|----------|------|------|
| `mem:{uid}:facts` | List | 用户事实/长期记忆（最多 30 条） |
| `mem:{uid}:summary` | String | 用户画像摘要（90 天过期） |
| `mem:{uid}:session_num` | String | 当前会话编号（自增） |
| `mem:{uid}:history` | List | 最近对话记录（最多 20 条） |

#### 核心方法

| 方法 | 功能 |
|------|------|
| `get_session_id()` | 获取当前会话 ID |
| `reset_session()` | 重置会话（递增编号，清除对话历史） |
| `save_turn()` | 保存一轮对话 |
| `get_recent_history()` | 获取最近 n 轮对话 |
| `add_fact()` | 添加用户事实 |
| `get_facts()` | 获取所有事实 |
| `set_summary()` | 设置用户画像摘要 |
| `get_summary()` | 获取用户画像摘要 |
| `build_context()` | 组装完整上下文（画像 + 事实 + 对话） |

---

### 7. `rag/info_index.py` - 知识库检索

| 组件 | 功能 |
|------|------|
| `InfoIndex` | 知识库检索类 |
| `hybrid_retriever` | Milvus 混合检索器（BM25 + 稠密向量） |
| `reranker_retriever` | BGE 重排序检索器 |
| `search()` | 执行混合检索 + 重排序 |

**检索流程**:
```
查询 → BM25 + 稠密向量混合检索 (top 10) → BGE Reranker 重排序 (top 3) → 返回结果
```

**技术栈**:
- 向量数据库: Milvus
- 嵌入模型: Ollama qwen3-embedding:0.6b
- 重排序模型: BAAI/bge-reranker-large
- 检索方式: BM25 + 稠密向量混合检索

---

### 8. `SDK/send_robot_private_message.py` - 消息发送 SDK

| 函数 | 功能 |
|------|------|
| `get_token()` | 获取 access_token（带缓存，2 小时有效，提前 200 秒刷新） |
| `send_robot_private_message()` | 发送机器人私聊消息 |
| `main()` | 独立运行模式（命令行） |

**Token 缓存机制**:
```python
_token_cache = {"token": None, "expire": 0}
```
- 有效期内直接返回缓存
- 过期后重新获取
- 提前 200 秒刷新
