# 小白机器人 - 项目功能总结

## 项目概述

「小白」是一个基于钉钉平台的 AI 聊天机器人，集成了 LangChain Agent、MiMo 大模型、Redis 记忆系统，支持日程管理、签到查询、消息发送等功能。

---

## 目录结构

```
scripts/
├── main.py                          # 启动入口
├── config.py                        # 配置文件
├── __init__.py                      # 包初始化
├── tools/
│   ├── __init__.py                  # 工具包初始化
│   ├── agent.py                     # LangChain Agent 模块
│   ├── tools.py                     # 工具定义（11个工具）
│   ├── processMsg.py                # 钉钉消息处理器
│   └── memory_store.py              # Redis 记忆存储
└── SDK/
    ├── __init__.py                  # SDK 包初始化
    └── send_robot_private_message.py # 钉钉消息发送 SDK
```

## 文件功能详解

### 1. `main.py` - 启动入口

| 功能 | 说明 |
|------|------|
| 日志初始化 | 配置日志格式和输出 |
| 钉钉连接 | 使用 `dingtalk_stream` 建立长连接 |
| 消息注册 | 注册 `PrivateMessageHandler` 处理私聊消息 |
| 永久运行 | `start_forever()` 保持服务常驻 |

**启动命令**: `python main.py`

---

### 2. `config.py` - 配置文件

| 配置项 | 说明 |
|--------|------|
| `llm_id` | 模型选择开关（0=MiMo API, 1=本地 Ollama） |
| `llm` | LLM 模型实例（MiMo 或 qwen2.5） |
| `redis_url` | Redis 连接地址 |
| `system_text` | 系统提示词（小白人设 + 工具说明） |
| `client_id` | 钉钉应用 AppKey |
| `client_secret` | 钉钉应用 AppSecret |
| `accessToken` | 钉钉访问令牌（动态刷新） |
| `userid` | 用户 ID |
| `unionid` | 用户 UnionID（日程 API 需要） |
| `robot_code` | 机器人代码 |

**LLM 切换**:
- `llm_id=0`: MiMo API（`https://token-plan-cn.xiaomimimo.com/v1`）
- `llm_id=1`: 本地 Ollama（`qwen2.5`）

---

### 3. `tools/agent.py` - LangChain Agent 模块

| 组件 | 功能 |
|------|------|
| `MemoryStore` | 初始化 Redis 记忆系统 |
| `create_tools()` | 创建所有工具实例 |
| `ChatPromptTemplate` | 构建提示模板（系统提示 + 聊天历史 + 用户输入） |
| `create_tool_calling_agent()` | 创建 LangChain Agent |
| `AgentExecutor` | Agent 执行器（max_iterations=5） |
| `SimpleRedisChatMessageHistory` | Redis 聊天历史实现 |
| `RunnableWithMessageHistory` | 带历史记录的链 |
| `chat()` | 处理一轮对话的主函数 |

**调用流程**:
```
用户消息 → chat() → 获取记忆上下文 → Agent 处理 → 保存对话 → 返回回复
```

---

| 工具名 | 类型 | 功能 | API 端点 |

| `send_private_message` | StructuredTool | 发送钉钉私聊消息 | 钉钉 SDK |
| `remember_fact` | StructuredTool | 记住用户信息 | Redis |
| `recall_memory` | StructuredTool | 查询用户记忆 | Redis |
| `get_checkin_records` | StructuredTool | 获取签到记录 | `oapi.dingtalk.com/topapi/checkin/record/get` |
| `create_calendar_event` | StructuredTool | 创建日程 | `api.dingtalk.com/v1.0/calendar/.../events` |
| `update_calendar_event` | StructuredTool | 修改日程 | PUT `api.dingtalk.com/v1.0/calendar/.../events/{id}` |
| `get_calendar_event` | StructuredTool | 查询日程详情 | `api.dingtalk.com/v1.0/calendar/.../events/{id}` |

| 模型 | 字段 |
|------|------|
| `SendMessageInput` | user_id, content |

| `RecallMemoryInput` | user_id |
| `CheckinRecordsInput` | user_ids, start_time, end_time |
| `CreateCalendarInput` | summary, start_time, end_time, description, location |
| `UpdateCalendarInput` | event_id, summary, start_time, end_time, description, location |
| `GetCalendarInput` | event_id |
| `DeleteCalendarInput` | event_id |
| `ListCalendarInput` | start_time, end_time |

#### 公共函数

| 函数 | 功能 |
|------|------|
| `get_fresh_token()` | 获取新的 access_token 并更新到 config |

---

### 5. `tools/processMsg.py` - 消息处理器

| 组件 | 功能 |
|------|------|
| `PrivateMessageHandler` | 继承 `dingtalk_stream.ChatbotHandler` |
| `process()` | 处理私聊消息的异步方法 |

**处理流程**:
```
接收消息 → 提取 user_id 和 message_text → 调用 agent.chat() → 发送回复
```

**错误处理**: 如果处理失败，发送 "抱歉，处理消息时出现错误" 给用户。

---

### 6. `tools/memory_store.py` - 记忆存储

#### Redis 数据结构

| Key 格式 | 类型 | 说明 |
|----------|------|------|
| `chat:{uid}:recent` | List | 最近对话记录（最多 20 条） |
| `chat:{uid}:facts` | List | 用户事实/记忆（最多 30 条） |
| `chat:{uid}:summary` | String | 用户画像摘要（90 天过期） |

#### 核心方法

| 方法 | 功能 |
|------|------|
| `save_turn()` | 保存一轮对话（用户问 + AI 答） |
| `get_recent()` | 获取最近 n 轮对话 |
| `format_recent()` | 格式化为 prompt 可读的上下文 |
| `add_fact()` | 添加用户事实 |
| `get_facts()` | 获取所有事实 |
| `set_summary()` | 设置用户画像摘要 |
| `get_summary()` | 获取用户画像摘要 |
| `build_context()` | 组装完整上下文（画像 + 事实 + 对话） |

---

### 7. `SDK/send_robot_private_message.py` - 消息发送 SDK

| 函数 | 功能 |
|------|------|
| `get_token()` | 获取 access_token（带缓存，2 小时有效，提前 200 秒刷新） |
| `send_robot_private_message()` | 发送机器人私聊消息 |
| `main()` | 独立运行模式（命令行） |

**Token 缓存机制**:
```python
_token_cache = {"token": None, "expire": 0}
```
- 有效期内直接返回缓存
- 过期后重新获取
- 提前 200 秒刷新

## API 端点汇总

| API | 端点 | 方法 |
|-----|------|------|
| 获取 Token | `api.dingtalk.com/v1.0/oauth2/accessToken` | POST |
| 发送消息 | `api.dingtalk.com/v1.0/robot/oToMessages/batchSend` | POST |
| 签到记录 | `oapi.dingtalk.com/topapi/checkin/record/get` | POST |
| 创建日程 | `api.dingtalk.com/v1.0/calendar/users/{unionId}/calendars/primary/events` | POST |
| 修改日程 | `api.dingtalk.com/v1.0/calendar/users/{unionId}/calendars/primary/events/{eventId}` | PUT |
| 查询日程 | `api.dingtalk.com/v1.0/calendar/users/{unionId}/calendars/primary/events/{eventId}` | GET |
| 删除日程 | `api.dingtalk.com/v1.0/calendar/users/{unionId}/calendars/primary/events/{eventId}` | DELETE |
| 日程列表 | `api.dingtalk.com/v1.0/calendar/users/{unionId}/calendars/primary/events` | GET |

## 技术栈

| 组件 | 技术 |
|------|------|
| LLM | MiMo API / Ollama qwen2.5 |
| Agent 框架 | LangChain (langchain_classic) |
| 消息队列 | dingtalk_stream |
| 缓存/存储 | Redis |
| HTTP 客户端 | requests |
| 数据验证 | Pydantic |
| 钉钉 SDK | alibabacloud-dingtalk |
