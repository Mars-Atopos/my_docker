## 详细问题记录

### 1. Ollama qwen2.5 CUDA 错误

**现象**:
```
CUDA error: unknown error (status code: 500)
```

**原因**: 本地 GPU 驱动或 CUDA 版本与 Ollama 不兼容

**解决方案**: 切换到 MiMo API
```python
llm = ChatOpenAI(
    base_url="https://token-plan-cn.xiaomimimo.com/v1",
    api_key="...",
    model="xiaomi/mimo-v2.5-pro",
)
```

---

### 2. Tool 单参数限制

**现象**:
```
Too many arguments to single-input tool create_calendar_event.
Consider using StructuredTool instead.
```

**原因**: `Tool()` 包装的工具只支持单个字符串参数，LLM 尝试传多个参数

**解决方案**: 改用 `StructuredTool` + Pydantic 模型
```python
class CreateCalendarInput(BaseModel):
    summary: str = Field(description="日程标题")
    start_time: str = Field(description="开始时间")
    end_time: str = Field(description="结束时间")

StructuredTool.from_function(
    func=create_calendar_event,
    args_schema=CreateCalendarInput,
)
```

### 3. Python 2/3 兼容 - iteritems

**现象**:
```
'dict' object has no attribute 'iteritems'
```

**原因**: 淘宝 SDK 使用 Python2 语法

**解决方案**:
```python
# Python2
for key, value in dict.iteritems():
# Python3
for key, value in dict.items():
```

---

### 4. Python 2/3 兼容 - HTTPSConnection

**现象**:
```
bind(): AF_INET address must be tuple, not int
```

**原因**: Python3 移除了 `strict` 参数，参数位置变化

**解决方案**:
```python
# Python2
connection = httplib.HTTPSConnection(domain, port, None, None, False, timeout)
# Python3
connection = http.client.HTTPSConnection(domain, port, timeout=timeout)
```

---

### 5. Python 2/3 兼容 - bytes/str

**现象**:
```
can only concatenate str (not "bytes") to str
```

**原因**: `response.read()` 在 Python3 返回 bytes

**解决方案**:
```python
# Python2
result = response.read()
# Python3
result = response.read().decode('utf-8')
```

---

### 6. Python 2/3 兼容 - urllib

**现象**:
```
AttributeError: module 'urllib' has no attribute 'urlencode'
```

**原因**: Python3 将 `urllib` 拆分为子模块

**解决方案**:
```python
# Python2
import urllib
urllib.urlencode(params)
# Python3
import urllib.parse
urllib.parse.urlencode(params)
```

---

### 7. Python 2/3 兼容 - has_key

**现象**:
```
AttributeError: 'dict' object has no attribute 'has_key'
```

**解决方案**:
```python
# Python2
if dict.has_key(key):
# Python3
if key in dict:
```

---

### 8. Python 2/3 兼容 - unicode

**现象**:
```
NameError: name 'unicode' is not defined
```

**原因**: Python3 没有 `unicode` 类型，`str` 就是 Unicode

**解决方案**: 移除 `unicode` 相关检查

---

### 9. Python 2/3 兼容 - hashlib

**现象**:
```
TypeError: Unicode-objects must be encoded before hashing
```

**解决方案**:
```python
# Python2
hashlib.md5(parameters)
# Python3
hashlib.md5(parameters.encode('utf-8'))
```

### 10. 日程 API 端点错误

**现象**:
```json
{"code":"InvalidAction.NotFound","message":"Specified api is not found"}
```

**尝试过的端点**:
- `https://oapi.dingtalk.com/calendar/v2/event/create` ❌
- `https://api.dingtalk.com/v1.0/calendar/events` ❌
- `https://api.dingtalk.com/v1.0/calendar/users/{userId}/events` ❌

**正确端点**:
```
https://api.dingtalk.com/v1.0/calendar/users/{unionId}/calendars/primary/events
```

### 11. 日程创建缺少 timezone

**现象**:
```json
{"code":"invalidParameter","message":"Start timezone cannot be blank for a non-all day event."}
```

**解决方案**: 在 start/end 中添加 `timeZone`
```json
{
  "start": {
    "dateTime": "2026-07-13T09:00:00+08:00",
    "timeZone": "Asia/Shanghai"
  }
}
```

### 12. 日程修改方法错误

**现象**: PATCH 请求返回 404

**原因**: 钉钉日程修改 API 不支持 PATCH

**解决方案**:
1. 改用 PUT 方法
2. 请求体必须包含 `id` 字段

```python
resp = requests.put(url, json={"id": event_id, "summary": "新标题"}, headers=headers)
```

---

### 13. 钉钉消息格式错误

**现象**:
```json
{"code":"invalidParameter.msgParam.invalid","message":"msgParam格式不正确"}
```

**原因**: 回复内容包含 markdown 代码块和特殊字符

**解决方案**:
```python
# 清理特殊字符
clean_msg = msg.replace('```', '').replace('```json', '')
msg_param = json.dumps({"content": clean_msg}, ensure_ascii=False)
```

### 14. LLM 不调用工具

**现象**: LLM 输出 JSON 文本而非实际调用工具
```
```json
{"name": "create_calendar_event", "arguments": {...}}
```
```

**原因**: Ollama qwen2.5 不支持原生 function calling

**解决方案**:
1. 切换到支持 function calling 的模型（MiMo）
2. 或使用 ReAct 模式解析文本输出

### 15. LLM 日期年份错误

**现象**: LLM 生成 2025 年日期，实际是 2026 年

**原因**: 模型训练数据截止日期导致不知道当前年份

**解决方案**: 系统提示词动态注入当前日期
```python
from datetime import datetime
_current_date = datetime.now().strftime("%Y年%m月%d日")
system_text = f"当前日期：{_current_date}\n..."
```

### 17. Token 过期问题

**现象**: API 返回 500 或权限错误

**原因**: 硬编码的 `accessToken` 已过期

**解决方案**: 动态获取 token 并缓存
```python
def get_fresh_token():
    class Options:
        client_id = config.client_id
        client_secret = config.client_secret
    token = send_robot_private_message.get_token(Options())
    if token:
        config.accessToken = token
    return token
```

## 经验总结

### Python 2/3 迁移要点
1. `dict.iteritems()` → `dict.items()`
2. `dict.has_key(k)` → `k in dict`
3. `urllib.urlencode` → `urllib.parse.urlencode`
4. `response.read()` → `response.read().decode('utf-8')`
5. `httplib` → `http.client`
6. `unicode` → 移除
7. `hashlib.md5(str)` → `hashlib.md5(str.encode())`

### 钉钉 API 调用要点
1. 日程 API 用新版端点 `api.dingtalk.com`
2. 时间必须带 `timeZone`
3. 修改用 PUT，不用 PATCH
4. 请求体要包含 `id`
5. Token 要动态获取，不要硬编码

### LangChain 工具定义要点
1. 多参数工具用 `StructuredTool` + Pydantic
2. 不要用 `Tool()` 包装多参数函数
3. 模型要支持 function calling
4. 系统提示词要包含当前日期
