# 手写 Agent 流程 —— 结合函数的完整指南

## 总览：6 步流水线

写一个 LangChain Agent，就是按顺序组装这 6 个模块。每一步都有**固定的函数**要写：

```
步骤1: 定义 LLM          → config.py 里的 llm
步骤2: 定义记忆           → memory_store.py 里的 MemoryStore 类
步骤3: 定义工具           → tools.py 里的 create_tools() 函数
步骤4: 组装 Prompt        → agent.py 里的 ChatPromptTemplate
步骤5: 创建 Agent+执行器  → agent.py 里的 create_tool_calling_agent() + AgentExecutor
步骤6: 写 chat() 入口函数 → agent.py 里的 chat() 函数
```

---

## 步骤 1: 定义 LLM（config.py）

**写什么**：创建一个 LLM 实例，告诉 Agent "你的大脑是什么"。

```python
from langchain_openai import ChatOpenAI

llm = ChatOpenAI(
    base_url="https://你的API地址/v1",
    api_key="你的key",
    model="模型名",
)
```

**为什么**：Agent 的所有推理都靠这个 LLM。后面所有步骤都要传入这个 `llm`。

---

## 步骤 2: 定义记忆（memory_store.py）

**写什么**：一个类，负责把用户的对话历史和画像存到 Redis。

```python
import json
import redis
from datetime import datetime

class MemoryStore:
    def __init__(self):
        self.rc = redis.Redis.from_url("redis://localhost:6379/1", decode_responses=True)

    def _key(self, uid, tag):
        return f"chat:{uid}:{tag}"

    # ---- 存对话 ----
    def save_turn(self, uid, user_msg, ai_msg):
        """每轮对话结束后调用，保存问答对"""
        entry = json.dumps({"q": user_msg, "a": ai_msg, "t": datetime.now().strftime("%m-%d %H:%M")}, ensure_ascii=False)
        self.rc.rpush(self._key(uid, "recent"), entry)
        self.rc.ltrim(self._key(uid, "recent"), -20, -1)  # 只保留最近20条

    def format_recent(self, uid, n=10):
        """取出最近n轮，格式化成文本给 prompt 用"""
        raw = self.rc.lrange(self._key(uid, "recent"), -n, -1)
        if not raw:
            return ""
        lines = []
        for r in raw:
            h = json.loads(r)
            lines.append(f"[{h['t']}] 用户: {h['q']}")
            lines.append(f"[{h['t']}] 小白: {h['a']}")
        return "\n".join(lines)

    # ---- 存用户画像（facts） ----
    def add_fact(self, uid, fact):
        """Agent 主动调用，记住一条用户信息"""
        entry = json.dumps({"fact": fact, "t": datetime.now().strftime("%Y-%m-%d")}, ensure_ascii=False)
        self.rc.rpush(self._key(uid, "facts"), entry)
        self.rc.ltrim(self._key(uid, "facts"), -30, -1)

    def get_facts(self, uid):
        """取出所有事实"""
        raw = self.rc.lrange(self._key(uid, "facts"), 0, -1)
        return [json.loads(r)["fact"] for r in raw]

    # ---- 组装上下文（最关键的方法） ----
    def build_context(self, uid):
        """把所有记忆拼成一段文本，注入到 prompt"""
        parts = []
        facts = self.get_facts(uid)
        if facts:
            parts.append("【已知信息】\n" + "\n".join(f"- {f}" for f in facts))
        recent = self.format_recent(uid)
        if recent:
            parts.append(f"【最近对话】\n{recent}")
        return "\n\n".join(parts) if parts else ""
```

**为什么**：LLM 是无状态的，不记得上一轮说了什么。你需要：
- `save_turn()` → 每轮结束后存下来
- `build_context()` → 每轮开始前取出来，拼到消息里
- `add_fact()` / `get_facts()` → 让 Agent 主动记住用户信息

**核心理解**：`build_context()` 的输出会直接拼到用户消息前面，让 LLM "看到"历史。

## 步骤 3: 定义工具（tools.py）

**写什么**：告诉 Agent "你能做什么事"。每个工具是一个 Python 函数 + 描述。

```python
from langchain_core.tools import Tool, StructuredTool
from pydantic import BaseModel, Field

# 多参数工具需要定义输入模型
class RememberFactInput(BaseModel):
    user_id: str = Field(description="用户ID")
    fact: str = Field(description="要记住的事实内容")

def create_tools(memory):
    """工厂函数，接收 memory 实例，返回工具列表"""

    # 工具1：搜索
    def search_web(query: str) -> str:
        """搜索互联网"""
        # ... 调用搜索API ...
        return "搜索结果"

    # 工具2：记忆（需要用到 memory）
    def remember_fact(user_id: str, fact: str) -> str:
        memory.add_fact(user_id, fact)
        return f"已记住: {fact}"

    # 返回工具列表
    return [
        Tool(name="search_web", func=search_web,
             description="搜索互联网获取最新信息"),  # 单参数用 Tool
        StructuredTool.from_function(
            func=remember_fact,
            name="remember_fact",
            description="记住关于用户的重要信息",
            args_schema=RememberFactInput,  # 多参数用 StructuredTool + Pydantic
        ),
    ]
```

**为什么**：
- `description` 非常重要！LLM 靠它决定什么时候调用哪个工具
- 工具函数的**返回值**会作为上下文传回给 LLM，所以要返回有意义的字符串
- 用工厂函数 `create_tools(memory)` 是为了让工具能访问 memory（闭包）

**两种写法选择**：
```
单参数 → Tool(name=..., func=..., description=...)
多参数 → StructuredTool.from_function(func=..., args_schema=PydanticModel, ...)
```

## 步骤 4: 组装 Prompt（agent.py）

**写什么**：定义发给 LLM 的消息模板。

```python
from langchain_core.prompts import ChatPromptTemplate, MessagesPlaceholder

prompt = ChatPromptTemplate.from_messages([
    ("system", "你是一个AI助手..."),           # ① 系统人设
    MessagesPlaceholder("chat_history"),       # ② 聊天历史（框架自动填）
    ("user", "{input}"),                       # ③ 用户消息（build_context 的结果会拼在这里）
    MessagesPlaceholder("agent_scratchpad"),   # ④ Agent 的思考过程（框架自动填）
])
```

**为什么这样排**：
- ① 告诉 LLM "你是谁"
- ② 框架自动把历史消息填进去（第一层记忆）
- ③ 用户的真实消息（包含第二层记忆 build_context 的结果）
- ④ Agent 调用工具时的中间思考过程

**注意**：`{input}` 是唯一的用户输入变量，`chat_history` 和 `agent_scratchpad` 是框架保留的。

## 步骤 5: 创建 Agent + 执行器（agent.py）

**写什么**：把 LLM + 工具 + Prompt 组装成一个可运行的 Agent。

```python
from langchain_classic.agents import AgentExecutor, create_tool_calling_agent

# 创建 Agent（LLM + 工具 + Prompt 的组合）
agent = create_tool_calling_agent(llm, tools, prompt)

# 创建执行器（负责实际运行 Agent，处理工具调用循环）
executor = AgentExecutor(
    agent=agent,
    tools=tools,
    handle_parsing_errors=True,  # 出错时自动重试
    max_iterations=5,            # 最多调用5次工具，防止死循环
)
```

**为什么分两层**：
- `agent` 只负责"思考"——决定调用哪个工具、怎么回复
- `executor` 负责"执行"——实际调用工具、处理错误、控制循环次数

## 步骤 6: 写 chat() 入口函数（agent.py）

**写什么**：把上面所有组件串起来，处理一轮完整对话。

```python
# 聊天历史管理（第一层记忆）
class SimpleRedisChatMessageHistory(BaseChatMessageHistory):
    # ... （见你的 agent.py:54-115，这段代码是固定的，直接复用）

def get_session_history(session_id):
    return SimpleRedisChatMessageHistory(session_id=session_id, redis_url=config.redis_url)

# 把执行器包装成带历史记录的链
chain_with_history = RunnableWithMessageHistory(
    runnable=executor,
    get_session_history=get_session_history,
    input_messages_key="input",
    history_messages_key="chat_history",
)

def chat(user_id, message):
    """处理一轮对话 —— 这是唯一的对外接口"""

    # 1. 取出第二层记忆（用户画像+历史对话），拼到消息前面
    memory_context = memory.build_context(user_id)
    if memory_context:
        full_message = f"{memory_context}\n\n用户说: {message}"
    else:
        full_message = message

    # 2. 调用 Agent（第一层记忆由 RunnableWithMessageHistory 自动处理）
    result = chain_with_history.invoke(
        {"input": full_message},
        config={"configurable": {"session_id": user_id}}
    )

    # 3. 提取回复
    reply = result["output"]

    # 4. 保存本轮对话到第二层记忆
    memory.save_turn(user_id, message, reply)

    return reply
```

**这个函数做了 4 件事**：
```
① build_context()  → 取出记忆，拼到消息前面
② invoke()         → 调用 Agent（自动带上聊天历史）
③ 取 reply         → 拿到 Agent 的回复
④ save_turn()      → 把本轮问答存到记忆
```

## 完整调用链（一张图）

```
用户发消息
    │
    ▼
chat(user_id, message)                    ← 你写的入口
    │
    ├─① memory.build_context(user_id)     ← 取出记忆，拼到消息前面
    │
    ├─② chain_with_history.invoke()       ← 调用 Agent
    │      │
    │      ├─ Prompt 模板组装:
    │      │    system + chat_history(自动) + input(含记忆) + scratchpad(自动)
    │      │
    │      ├─ LLM 推理: 决定回复还是调用工具
    │      │
    │      ├─ 如果调用工具 → 执行工具函数 → 结果返回 LLM → 继续推理
    │      │   （最多循环 max_iterations 次）
    │      │
    │      └─ 返回 result["output"]
    │
    ├─③ 提取 reply
    │
    └─④ memory.save_turn(uid, msg, reply) ← 存储本轮对话
    │
    ▼
返回 reply
```

## 最小可运行模板

如果你要从零手写一个 Agent，只需要这 4 个文件：

```
my_agent/
├── config.py          # 步骤1: LLM配置
├── memory_store.py    # 步骤2: 记忆类
├── tools.py           # 步骤3: 工具定义
└── agent.py           # 步骤4-6: Prompt + Agent + chat()
```

每个文件写什么、为什么写，上面都列清楚了。**记忆部分只需要记住**：
- `save_turn()` → 存（每轮结束调一次）
- `build_context()` → 取（每轮开始调一次）
- `add_fact()` → 让 Agent 主动记（通过工具）
